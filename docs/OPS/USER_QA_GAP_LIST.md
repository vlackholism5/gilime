# User 미완료 QA 갭 목록 (CTO 회의 아젠다 1 산출)

**목적:** 사용자 화면(홈, 전체 이슈, 경로 찾기, 마이노선·routes·alerts)에서 "작업이 안 되어 있는" QA 항목을 페이지별·항목별로 식별하고 우선순위를 부여. 회의 Build 목록 B1과 연동.

**기준:** CTO 회의 결론 플랜(cto_회의_결론_build_실행), GAP LIST 요약 표.

---

## 갭 목록

| 페이지 | 갭 설명 | 우선순위 | 검증 방법 |
|--------|---------|----------|-----------|
| 홈 | 빈 이슈 시 empty state 문구 일관성, 긴급 이슈 Top3·CTA 동선 | P2 | 이슈 0건 시 메시지·CTA 클릭 흐름 확인 |
| 전체 이슈 | 이슈 0건 시 "이슈가 없습니다" 노출, 필터/정렬·데이터 연동 검증 항목 없음 | P1 | 필터(전체/긴급/운행중단/행사/공지)·정렬(영향도순/최신순)·페이징 스모크 |
| 경로 찾기 | Phase-2 미구현: 안내시작 비활성, step=recommend/subscribed 카드 "Phase 2" 문구만 | P2 | Phase-2 LOCK 유지, 예시 링크·구독 안내 동작 확인 |
| 마이노선(my_routes) | 구독 0건/알림 없음 empty state, CTA(노선 구독 관리·구독 노선 알림 보기) | P2 | empty state 문구·CTA 링크 목적지 확인 |
| routes.php | 노선 컬럼이 route_label 기반 — "운행구간" 등 동일 라벨 반복 시 식별 어려움 | P2 | 목록 스모크, 문서화 또는 라벨 보강 추후 |
| alerts.php | 구독 노선 알림 empty state "알림이 없습니다", 유형 필터·구독 노선만 해제 | P2 | empty state·필터 동작 확인 |

---

## 우선순위 정의

- **P0:** 사용자/운영에 즉시 영향, 반드시 회의 후 빌드에 포함. (본 문서 신규 작성으로 User QA 갭 목록 부재 P0 해소.)
- **P1:** 단기 내 처리 권장, 빌드 목록에 포함 가능.
- **P2:** 문서화 또는 추후 스프린트.
