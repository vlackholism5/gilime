# v1.5-03 Smoke Checklist (Delivery semantics + pagination)

## 1) Pagination UI

- /user/alerts.php 접속. 하단에 "Page 1 (50 per page)", Next 링크(이벤트가 50건 이상일 때) 표시 확인.
- `?page=2` 로 이동 시 2페이지 내용 표시, Previous/Next 동작 확인.
- type, route_label, subscribed 필터 적용 후 page 파라미터가 유지되는지 확인.

## 2) Deliveries only for rendered page

- (선택) 이벤트가 50건을 넘도록 E2E 방식으로 테스트 이벤트 추가. 50건 미만이면 "확인 필요"로 남기고 1페이지만 확인.
- page=1 접속 → app_alert_deliveries 건수 기록.
- page=2 접속 → 2페이지에만 보이는 이벤트에 대해서만 delivery가 추가되는지(건수 증가가 2페이지 이벤트 수 이하인지) 확인.
- page=1 재접속(새로고침): **같은 이벤트**에 대한 delivery는 이미 있으므로, UNIQUE로 인해 건수 증가 없어야 함.

## 3) Refresh does not grow count (same page)

- 특정 페이지(예: page=1) 여러 번 새로고침.
- `SELECT COUNT(*) FROM app_alert_deliveries` 반복 실행. 동일 사용자·동일 페이지에서는 증가하지 않아야 함.
