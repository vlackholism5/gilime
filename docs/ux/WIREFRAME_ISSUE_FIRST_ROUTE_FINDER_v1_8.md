# Wireframe v1.8 — 이슈+길찾기 듀얼 진입 UX

## 1. 목표/원칙 (3줄)

- 사용자가 메인에서 "길찾기 서비스"로 즉시 인식되는 User UI를 구성한다.
- 이슈가 여러 건이어도 메인은 길찾기 입력(출발/도착)을 우선 노출하고, 이슈는 요약 카드로만 표시한다.
- 사용자는 (A) 길찾기부터 시작하거나 (B) 이슈를 클릭해 이슈 컨텍스트 기반 길찾기를 시작할 수 있는 듀얼 진입을 지원한다.

## 3. 메인 홈(U-HOME) 와이어프레임(ASCII)

모바일 1열 기준.

```
+------------------------------------------+
| 길라임          홈  노선  알림  길찾기    |
+------------------------------------------+
| [1번 영역] 출발/도착 입력                 |
| ---------------------------------------- |
| 출발지                                    |
| [___________________________]             |
| 도착지                                    |
| [___________________________]             |
| [  경로 찾기  ]                           |
+------------------------------------------+
| [2번 영역] 긴급 이슈 Top3                 |
| ---------------------------------------- |
| OO 파업 (Active) High                     |
| 임시 셔틀 17개 노선 운영중                |
| [이슈 보기] [이슈 기반 길찾기]            |
| ---------------------------------------- |
| XX선 운행중단 (Active) Medium             |
| 해당 구간 버스 운행 중단                  |
| [이슈 보기] [이슈 기반 길찾기]            |
| ---------------------------------------- |
| 설 연휴 배차 변경 (Ended) Low             |
| 2/9~2/12 배차 조정                        |
| [이슈 보기] [이슈 기반 길찾기]            |
| ---------------------------------------- |
| [전체 이슈 보기]                          |
+------------------------------------------+
```

- 상단 1번: 출발/도착 입력 + [경로 찾기] 버튼
- 하단 2번: 긴급 이슈 Top3 카드 (상태, 영향도, CTA 2개, [전체 이슈 보기])
- CTA 우선순위: [이슈 기반 길찾기]를 Primary로 강조 (btn-gilaime-primary)
- 이슈 0건 시: "현재 긴급 이슈 없음" 문구 + [전체 이슈 보기]만 표시
- (선택) 활성 이슈 상단에 긴급 배너: "OO 파업 중 — 임시 셔틀 17개 노선 운영"

## 6. 길찾기 입력(U-JNY-01) 와이어프레임(ASCII)

### 6.1 기본 (직입력, 현재)

```
+------------------------------------------+
| 길라임     홈  노선  알림  길찾기    [X] |
+------------------------------------------+
| 경로 찾기                                 |
| ---------------------------------------- |
| 출발지                                    |
| [___________________________]             |
| 도착지                                    |
| [___________________________]             |
| [ ] 임시 셔틀 포함                        |
| ---------------------------------------- |
| [  경로 찾기  ]                           |
+------------------------------------------+
```

- 출발지/도착지 입력
- [임시 셔틀 포함] 체크
- [경로 찾기] 버튼
- **그리드:** 상단 여백 --g-space-3, 카드 내부 --g-space-2 (NAVER_MAP_GRID_GUIDE)

### 6.2 Phase 2 확장 (네이버 지도 참조)

```
+------------------------------------------+
| [←] 검색 [_________________] [🎤] [X]   |
+------------------------------------------+
| [탭] 장소 | 버스정류장                    |
+------------------------------------------+
| [리스트] (검색 결과)                      |
| ---------------------------------------- |
| 문래역 2호선 · 735m                       |
| 서울 영등포구 당산로 지하 28    [선택]    |
| ---------------------------------------- |
| 문래역1번출구 정류장 · 699m               |
| 19318 문래우체국 방면        [선택]       |
| ---------------------------------------- |
+------------------------------------------+
```

- 검색바: 뒤로가기, 입력, 음성, X
- 탭: 장소 | 버스정류장 (검색 유형 전환)
- 리스트: 이름, 거리, 주소/방향, [선택]
- 버스정류장: ID, 방향, 노선 유형 뱃지
- **그리드:** 상단 여백 --g-space-3, 카드 내부 --g-space-2, 리스트 항목 간격 --g-space-2 (NAVER_MAP_GRID_GUIDE)

## 10. UX 규칙(이슈 여러 건일 때)

| 규칙 | 설명 |
|------|------|
| 메인 Top3 고정 | 이슈가 10개 이상이어도 메인은 Top3만 노출 |
| 메인 깨지지 않음 | 이슈 증가로 인해 메인 레이아웃/스크롤이 과도해지지 않음 |
| 전체 이슈 별도 화면 | U-ISSUES에서 필터/정렬 제공 |
| 이슈가 길찾기 밀어내기 금지 | 1번 영역(출발/도착)이 항상 상단 고정 |
| 이슈 요약 카드 | 메인에서는 제목·상태·영향도·CTA만, 본문은 상세 화면에서 |

## 11. 운영/심사 관점 포인트

| 항목 | 설득 이유 |
|------|-----------|
| 운영형 서비스 | 이슈·긴급 상황별 노출, 필터/정렬, 구독 반영 재탐색 등 운영·관리 관점이 드러남 |
| 이슈-길찾기 연계 | "이슈 기반 길찾기"로 이슈와 경로를 자연스럽게 연결, 사용자 니즈 충족 |
| 재현 가능한 흐름 | A: 길찾기 → 추천 → 구독 → 재탐색, B: 이슈 → 이슈 기반 길찾기 → 경로 |
| 차별성 | 기존 지도·공지 서비스와 달리 이슈와 경로를 하나의 흐름으로 통합 |

## 12. Gate(통과 조건)

| No | Gate | 확인 |
|----|------|------|
| 1 | U-HOME에서 길찾기 인식이 즉시 되는가 | 출발/도착 입력이 1번 영역에 노출 |
| 2 | 이슈 Top3가 존재해도 화면이 복잡하지 않은가 | 요약 카드만, CTA 2개로 제한 |
| 3 | 이슈가 10개여도 메인은 깨지지 않는가 | Top3 고정, 나머지는 [전체 이슈 보기] |
| 4 | 이슈 → 길찾기 흐름이 자연스러운가 | [이슈 기반 길찾기] CTA으로 이슈 상세에서 진입 |
| 5 | 길찾기 → 추천 → 구독 → 재탐색 흐름이 끊기지 않는가 | U-JNY-02 → U-JNY-03 → U-JNY-04 순서 |
| 6 | 구독이 경로에 반영되었다는 증거가 UI에 있는가 | "구독 반영 결과", "구독 노선" 문구 |
| 7 | 긴급상황에서 클릭 수가 과도하지 않은가 | 출발/도착 입력 → [경로 찾기] 2클릭 (v1.7 기준 3클릭 이내 충족) |
| 8 | 확인 필요 항목이 문서에 명시되었는가 | 섹션 13 |
| 9 | 터치 영역 44px 이상인가 | SOT_GILAIME_UI_SYSTEM 준수 |
| 10 | 키보드만으로 검색 가능한가 | 출발/도착 입력 포커스, Tab 이동 가능 |
