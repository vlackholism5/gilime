# 네이버 지도 UI/UX 도입 검토 — 멀티롤 회의 (v1.8)

## 1. 개요

길라임 MVP의 길찾기·경로 안내 화면에 **네이버 지도 UI/UX**를 공통 참조로 도입할 수 있는지, 멀티롤(Product · Tech · Design · UX) 관점에서 검토한 결과를 정리한다.

---

## 2. 네이버 지도 UI 패턴 요약

| 패턴 | 구성 요소 | 길라임 적용 가능 화면 |
|------|----------|------------------------|
| **장소/정류장 검색** | 검색바 + 탭(장소/버스정류장) + 지도 + 리스트(거리·주소·선택) | U-JNY-01 출발/도착 입력 |
| **경로 결과 탭** | 전체/버스/지하철/버스+지하철 필터 | U-JNY-02 경로 결과 |
| **경로 카드** | 소요시간·요금·타임라인 세그먼트·상세 단계·안내시작 | U-JNY-02, U-JNY-04 |
| **지도 뷰** | 경로 폴리라인·정류장 마커·지도중심/내위치 | Phase 2 지도 뷰 |

---

## 3. 멀티롤 검토 결과

### 3.1 Product

| 항목 | 판단 | 비고 |
|------|------|------|
| **공통 UI 도입 가능 여부** | ✅ 가능 | 장소 검색·경로 결과 형식은 사용자 학습 비용 낮음 |
| **와이어프레임 참조** | ✅ 권장 | U-JNY-01~04에 네이버 지도 패턴 반영 |
| **도입 범위** | 단계적 | Phase 1: 검색·결과 UI만 / Phase 2: 지도 뷰 |
| **차별화** | 유지 | 이슈-길찾기 통합, 임시 셔틀 뱃지 등 길라임 고유 요소 유지 |

### 3.2 Tech

| 항목 | 판단 | 비고 |
|------|------|------|
| **검색 UI 패턴** | ✅ 적용 가능 | 탭(장소/버스정류장) → 기존 `route_finder_resolve_stop` 확장 |
| **지도 API** | ⚠️ Phase 2 | 네이버/Kakao API 키·라이선스·배포 제한 검토 필요 |
| **경로 필터 탭** | ✅ 적용 가능 | 전체/버스/버스+임시셔틀 등 프론트만 변경 |
| **경로 카드 타임라인** | ✅ 적용 가능 | 세그먼트(버스·도보·임시셔틀) 시각화 |

### 3.3 Design

| 항목 | 판단 | 비고 |
|------|------|------|
| **SOT와의 정합성** | ✅ 유지 | `g-card`, `btn-gilaime-primary` 등 기존 토큰 유지 |
| **색상/아이콘** | 부분 차용 | 네이버 블루 대신 길라임 라임·회색 톤 유지 |
| **레이아웃** | 참조 | 상단 검색 → 중간 결과/지도 → 하단 리스트 구조 |
| **반응형** | 필수 | 모바일 우선, 터치 영역 44px 이상 |

### 3.4 UX

| 항목 | 판단 | 비고 |
|------|------|------|
| **3클릭 원칙** | ✅ 유지 | 출발/도착 + 검색 2클릭, 결과 즉시 노출 |
| **선택 플로우** | 동일 | 검색 → 리스트에서 "선택" → 경로 검색 (네이버와 동일) |
| **경로 없음 대안** | 유지 | [근처 정류장 보기] [임시 셔틀 추천] |
| **접근성** | 충족 | 키보드 포커스·단축키·텍스트 기반 안내 유지 |

## 4. 결론 및 권장안

### 4.1 결론

| 항목 | 결론 |
|------|------|
| **공통 UI 도입** | ✅ **가능** — 네이버 지도 패턴을 길라임 스타일로 변형해 적용 |
| **와이어프레임 참조** | ✅ **권장** — U-JNY-01~04 와이어프레임에 네이버 지도 패턴 반영 |
| **전체 설계** | ✅ **반영** — `WIREFRAME_ISSUE_FIRST_ROUTE_FINDER_v1_8.md`, `PLAN_UX_OPERATIONS_ROUTE_FINDER_v1_7.md`에 참조 추가 |

### 4.2 도입 단계

| 단계 | 범위 | 비고 |
|------|------|------|
| **즉시** | 검색 입력·결과 UI 패턴 참조 | 탭·경로 카드·필터 구조 |
| **Phase 1** | 장소/정류장 탭, 경로 필터 탭, 경로 카드 타임라인 | API 없이 UI만 |
| **Phase 2** | 지도 뷰 | 네이버/Kakao API 연동 |

### 4.3 제약·리스크

| 리스크 | 대응 |
|--------|------|
| 지도 API 비용·라이선스 | Phase 2까지 유보, MVP는 경로 목록만 |
| 네이버 UI 과도한 모방 | 색상·아이콘은 길라임 SOT 유지 |
| 정류장 검색 품질 | `seoul_bus_stop_master` 기반, 추후 Geocoding 확장 검토 |

## 5. 와이어프레임 참조 매핑

### 5.1 네이버 지도 → 길라임 화면 매핑

| 네이버 지도 화면 | 길라임 화면 | 참조할 패턴 |
|------------------|-------------|-------------|
| 장소 검색 (장소 탭) | U-HOME, U-JNY-01 출발지 | 검색바 + 탭 + 리스트 + [선택] |
| 버스정류장 검색 | U-JNY-01 출발/도착 | 정류장 ID·방향·거리·[선택] |
| 경로 결과 (전체/버스/지하철 탭) | U-JNY-02 | 전체/버스/버스+임시셔틀 탭 |
| 경로 카드 (33분, 타임라인) | U-JNY-02, U-JNY-04 | 소요시간·세그먼트 바·상세 단계 |
| 지도 + 경로 폴리라인 | U-RF-04 (Phase 2) | 지도 뷰 |

### 5.2 공통 컴포넌트 추출

| 컴포넌트 | 네이버 | 길라임 SOT |
|----------|--------|------------|
| 검색 탭 | 장소 \| 버스정류장 | `btn btn-sm` 토글 |
| 결과 리스트 항목 | 거리·주소·[선택] | `card g-card` 또는 `list-group` |
| 경로 필터 탭 | 전체 15 \| 버스 6 \| ... | `btn btn-sm` (issues.php 필터와 동일) |
| 경로 카드 | 소요시간·타임라인·단계 | `card g-card` + `progress` 또는 custom |
| [선택] 버튼 | 파란색 | `btn btn-gilaime-primary btn-sm` |
