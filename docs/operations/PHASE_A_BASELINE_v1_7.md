# Phase A: 기준선 고정 (2026-02-20 MVP)

## 1. MVP 범위 확정

**성공 기준:** "서울 1개 이슈 완주 데모"

### DoD (Definition of Done)
1. 관리자에서 PDF 1건 업로드 후 `source_doc` 생성 확인
2. 관리자에서 파싱/매칭 실행 후 후보 생성 확인
3. 관리자에서 노선 검수(승인/거절/별칭) 후 승격 실행 확인
4. 사용자에서 구독 노선 기준 경로 안내(정류장 순서) 확인
5. 감사 화면에서 전달 이력/상태 조회 확인

### In Scope
- 서울 1개 이슈 기준의 운영형 데모
- 관리자: 업로드, 파싱/매칭 실행, 노선 검수, 승격, 감사 조회
- 사용자: 노선 구독, 알림 확인, 구독 노선 경로 안내, 이슈 기반 길찾기

### Out of Scope
- OCR 자동화 구현 (설계 문서만 유지)
- 워커/큐 기반 비동기 파이프라인
- 실시간 교통/혼잡 최적화
- 대규모 지역 확장(서울 외)

---

## 2. 지도/경로 정의 고정

**결정:** MVP 단계에서는 **외부 길찾기 연동 수준**으로 고정.

| 항목 | MVP 범위 | 후속 단계 |
|------|----------|-----------|
| 경로 안내 | 구독 노선의 승격된 정류장 순서 제공 | - |
| 외부 길찾기 | 연동 없음. 텍스트/정류장 순서만 | 네이버/카카오 등 API 연동 |
| 지도 폴리라인 | 표시하지 않음 | 지도 위 경로 표시 |
| 셔틀 구간 | 텍스트 우선 (구간/환승 안내) | 시각화 고도화 |

경로 안내는 `route_finder.php` 및 `journey.php`에서 **정류장 순서 + 노선 정보**를 우선 제공한다.

## 3. 증거 수집 템플릿 고정

작업 단위마다 **브리프 + 게이트 리포트** 동시 운영.

### 브리프 템플릿 (PM_SYNC_TEMPLATE.md)
```
Done:
- 오늘 완료한 항목 3~5개

Evidence:
- 코드/문서 파일 경로
- 검증 결과(php -l, smoke, SQL 확인 결과)
- 화면 캡처 경로

Risk:
- 현재 리스크 1~3개

Next(24h):
- 다음 24시간 실행 항목 2~4개
```

### 게이트 리포트 템플릿
```
Gate ID: Gate-XX
Scope: (이번 게이트 대상 기능)

Checklist:
- [ ] 요구사항 충족
- [ ] 회귀 영향 점검
- [ ] 예외 시나리오 점검
- [ ] 증거 파일 첨부

Evidence Links:
- 코드: ...
- 문서: ...
- 로그/쿼리: ...
- 캡처: ...

Result: PASS / HOLD / FAIL
Decision Note: (다음 단계 진입 조건)
```
