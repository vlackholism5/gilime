# MVP Goal 2026-02-20

## 목적
- 2026-02-20까지 길라임 MVP를 "1개 이슈 완주 데모" 기준으로 확정한다.
- 기존 운영 파이프라인(업로드 → 파싱/매칭 → 검수 → 승격)을 재사용하고, 사용자 체감 기능을 보강한다.

## 성공 기준 (DoD)
1. 관리자에서 PDF 1건 업로드 후 `source_doc` 생성 확인.
2. 관리자에서 파싱/매칭 실행 후 후보 생성 확인.
3. 관리자에서 노선 검수(승인/거절/별칭) 후 승격 실행 확인.
4. 사용자에서 구독 노선 기준 경로 안내(정류장 순서) 확인.
5. 감사 화면에서 전달 이력/상태 조회 확인.

## 범위 (In)
- 서울 1개 이슈 기준의 운영형 데모.
- 관리자: 업로드, 파싱/매칭 실행, 노선 검수, 승격, 감사 조회.
- 사용자: 노선 구독, 알림 확인, 구독 노선 경로 안내.
- 문서: Gate/E2E/라이프사이클/재시도 정책/CRUD/UML 최소 세트.

## 비범위 (Out)
- OCR 자동화 구현 (설계 문서만 유지).
- 워커/큐 기반 비동기 파이프라인.
- 실시간 교통/혼잡 최적화.
- 대규모 지역 확장(서울 외).

## 지도/경로 정의 (MVP)
- 경로 안내는 "구독 노선의 승격된 정류장 순서"를 우선 제공한다.
- 외부 길찾기 API 연동/폴리라인은 후속 단계(확장 항목)로 분리한다.

## 운영 원칙
- 대규모 재구축 금지, 기존 파이프라인 확장 우선.
- 파일별 diff + `php -l` + smoke 체크를 기본 게이트로 사용.
- DB 변경은 문서화 후 승인 단위로만 실행한다.

## 증거 수집 규칙
- 작업 단위마다 브리프 + 게이트 리포트를 동시 작성한다.
- 증거는 파일 경로, 화면 캡처, 검증 쿼리 결과를 포함한다.
