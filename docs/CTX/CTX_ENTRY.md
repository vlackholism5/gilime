# CTX_ENTRY — GILIME (Token-Safe) v1.0

## 목적
- LLM(예: Gemini)에게 "프로젝트 전체를 읽지 못하게" 하고, 필요한 파일만 지정해 토큰 사용을 통제한다.
- 작업은 항상 "최소 파일 단위"로 진행한다.

## 절대 규칙
1) 폴더 단위/레포 전체 읽기 금지
2) 한 번 요청에서 읽을 파일은 최대 3개 (초과 시 쪼개서 진행)
3) 대용량/런타임/업로드/시크릿 경로는 절대 읽지 말 것 (BLOCKLIST 참고)
4) 먼저 요구사항을 "작업 단위"로 재정의한 뒤, 필요한 파일만 지정할 것
5) 결과는 반드시 diff 또는 파일별 변경 요약으로만 제시 (전체 파일 재출력 금지)
