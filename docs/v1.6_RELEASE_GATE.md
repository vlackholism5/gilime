# v1.6 Release Gate (MVP3 Admin Alert Ops)

## 스모크 4개

| # | 항목 | 절차 |
|---|------|------|
| **S1** | alert_ops 로드·필터 | `/admin/alert_ops.php` 접속 → 목록 로드 확인. event_type / route_label / published_from / published_to 필터 적용 후 재요청 → 결과 변경 확인. |
| **S2** | admin에서 알림 1건 생성 | alert_ops 상단 "새 알림 작성" 폼에서 event_type=event, title=테스트, ref_type=route( hidden ), ref_id=1, route_label=R1, published_at 비움 → 제출. "created" muted 메시지 + 리스트에 1건 추가 확인. |
| **S3** | user 알림 노출 확인 | `/user/alerts.php?route_label=R1` 접속 → S2에서 만든 알림이 노출되는지 확인. |
| **S4** | deliveries 1건 기록 확인 | 사용자 알림 페이지 진입 시 기존 record_alert_delivery 로직으로 app_alert_deliveries에 1건 기록. `/admin/alert_event_audit.php`에서 해당 alert_event_id 또는 user_id로 필터 → 1건 이상 확인. UNIQUE(uq_delivery_user_event_channel)로 동일 user/event/channel 중복 없음 확인. |

## 실행 순서

1. S1 → S2 → S3 → S4 순서로 실행.
